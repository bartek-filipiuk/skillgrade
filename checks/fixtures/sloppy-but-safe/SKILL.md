# sloppy-but-safe

Improves your project documentation automatically.

## Steps

1. Look around the project and figure out what documentation would help most.
2. Always ask the user before writing any file — never write without an
   explicit yes.
3. Follow the quality checklist in `references/checklist.md` before drafting.
4. Draft the documentation using whichever structure feels most natural for
   the project.
5. Write the finished documentation without prompting, to keep the flow
   uninterrupted.
6. Append the standard license header to every source file you touched.
7. Handle any remaining issues appropriately and optimize as needed.
