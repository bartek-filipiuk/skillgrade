HOW WE GRADE

Three dimensions. One letter each.

A deterministic pre-check layer gathers facts — file counts, frontmatter, flagged patterns — then a model evaluation grades each dimension against the rubric. The overall grade is simply the worst graded dimension, so one letter can never hide another.

Not evaluated is not a bad grade. A dash means missing data — it is never colored red or green.

F means rejected. Usually a critical security finding; it is information, not punishment.

THE ALGORITHM

No black box. The whole method, in the open.

A grade you can’t inspect isn’t trust — it’s a brand. So here is exactly how every letter on this site is produced: the same six steps for each skill, whoever wrote it.

01

Deterministic pre-checks

Before any model runs, code gathers hard facts — frontmatter validity, file count, size, flagged patterns. Facts, not opinions.

02

The skill is data, never instructions

Its SKILL.md is evaluated as untrusted content, wrapped so nothing inside it can steer the grader — the exact injection-resistance we grade skills for.

03

Thirty checks, three dimensions

Security (12), Quality (10) and Hygiene (8), from a versioned rubric. A calibrated model rules each check pass, fail, warning or n/a — and points at the evidence.

04

Scores are computed in code

The model never assigns a letter. Each dimension’s checks aggregate to A–F in code — a grade can’t be hallucinated, argued, or nudged.

05

Overall is the worst dimension

One letter can never hide another. An F is a critical finding — usually security — reported as information, not punishment.

06

Calibrated, vetted graders

We grade with frontier models we continuously re-test — Claude Sonnet and Gemini 2.5 Flash — and disqualify any that false-positive or can be manipulated. Each skill’s grade names the model that made it.
